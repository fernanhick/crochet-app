## 1. Background & Strategy

AI-generated content has uncertain copyright status in most jurisdictions. Courts have generally ruled that purely AI-generated work cannot be copyrighted by anyone. This means:

- Users cannot claim copyright over patterns generated by the app
- Anthropic/OpenAI retain no copyright either
- The app itself has no copyright claim over raw AI output

**What IS protectable:**
- The prompting system and output template (proprietary, server-side, never exposed)
- The brand and platform (Crochet Blueprint name, logo, UI)
- The Terms of Service contract between the platform and users

**Protection strategy = three layers:**
1. ToS clause — contractual personal-use-only restriction
2. In-pattern text watermark — brand embedded in stitch content at generation time
3. PDF watermark — diagonal overlay on exported PDFs

None of these prevent copying entirely. Together they create a clear paper trail of intent, make violations obvious, and give grounds for platform takedown requests (Etsy, Ravelry, etc).

## 3. In-Pattern Text Watermark

### 3.1 What it is

A plain-text attribution line baked into the generated pattern content by the Convex Action at generation time — before the pattern is stored or shown to the user. It is part of the stitch content, not a UI overlay, so it persists in any copy-paste, screenshot, or export of the pattern text.

### 3.2 Where to inject it

Inject the watermark in **two places** inside the pattern output:

**Place 1 — Pattern header block**  
At the very top of the generated pattern, before the materials list:

```
Generated by Crochet Blueprint — crochetblueprint.app
Personal use only. Not for resale. See full terms at crochetblueprint.app/terms
─────────────────────────────────────────────────────
```

**Place 2 — Pattern footer block**  
At the very end of the pattern, after the assembly section:

```
─────────────────────────────────────────────────────
© Crochet Blueprint — crochetblueprint.app
This pattern was AI-generated for personal use only.
Pattern ID: [PATTERN_ID] | Generated: [DATE] | User: [USER_ID_HASH]
```

The Pattern ID, date, and hashed user ID are logged to the Convex database at generation time. This creates a traceable record: if a pattern appears for sale elsewhere, you can match it back to the generating account.

### 3.3 Implementation — Convex Action change

In the `generatePattern` Convex Action, after the AI response is validated and parsed, inject the watermark strings before storing to the database:

```typescript
// After validatePatternText() passes and sections are parsed

const patternId = generatePatternId(); // e.g. nanoid(10)
const userIdHash = hashUserId(args.userId); // sha256 first 8 chars, not reversible
const generatedDate = new Date().toISOString().split('T')[0]; // YYYY-MM-DD

const header = [
  `Generated by Crochet Blueprint — crochetblueprint.app`,
  `Personal use only. Not for resale. See terms at crochetblueprint.app/terms`,
  `─────────────────────────────────────────────────────`,
  ``
].join('\n');

const footer = [
  ``,
  `─────────────────────────────────────────────────────`,
  `© Crochet Blueprint — crochetblueprint.app`,
  `This pattern was AI-generated for personal use only.`,
  `Pattern ID: ${patternId} | Generated: ${generatedDate} | User: ${userIdHash}`
].join('\n');

const wateredPatternText = header + parsedPatternText + footer;

// Store wateredPatternText (not the raw parsedPatternText) to the database
await ctx.db.insert('patterns', {
  userId: args.userId,
  patternId,
  text: wateredPatternText,
  // ... other fields
});
```

## 4. PDF Watermark

### 4.1 What it is

A diagonal semi-transparent text watermark rendered behind the pattern content on every page of the exported PDF. Unlike image watermarks, this is rendered at PDF generation time as a vector text element — it cannot be "erased" without reconstructing the PDF from scratch.

### 4.2 Visual spec

```
Text:        CROCHET BLUEPRINT — PERSONAL USE ONLY
Font:        Helvetica Bold (or DejaVu Sans Bold)
Size:        42pt
Colour:      #1A1A2E at 8% opacity (very subtle, readable content above it)
Rotation:    -35 degrees
Position:    Centred on each page
Repeat:      Once per page
```

8% opacity means it is barely visible on-screen but clearly shows on print and is unambiguous in the PDF source. It does not interfere with reading the pattern.

### 4.3 Implementation with pdf-lib

The PDF export uses `pdf-lib` (already a standard choice for React Native PDF generation). Add the watermark after all content is written, in a post-processing pass over every page:

```typescript
import { PDFDocument, rgb, degrees } from 'pdf-lib';
import { FontStyle } from 'pdf-lib'; // for font embedding

async function addWatermarkToPdf(pdfBytes: Uint8Array): Promise<Uint8Array> {
  const pdfDoc = await PDFDocument.load(pdfBytes);
  const pages = pdfDoc.getPages();
  const font = await pdfDoc.embedFont('Helvetica-Bold');

  const watermarkText = 'CROCHET BLUEPRINT — PERSONAL USE ONLY';
  const fontSize = 42;
  const textWidth = font.widthOfTextAtSize(watermarkText, fontSize);

  for (const page of pages) {
    const { width, height } = page.getSize();

    page.drawText(watermarkText, {
      x: width / 2 - textWidth / 2,
      y: height / 2,
      size: fontSize,
      font,
      color: rgb(0.102, 0.102, 0.180), // #1A1A2E
      opacity: 0.08,
      rotate: degrees(-35),
    });
  }

  return await pdfDoc.save();
}
```

Call `addWatermarkToPdf()` on the final PDF bytes before returning them to the client for download.

### 4.4 PDF footer line

In addition to the diagonal watermark, add a small footer line at the bottom of every PDF page:

```
Generated by Crochet Blueprint (crochetblueprint.app) · Pattern ID: [ID] · Personal use only
```

This mirrors the in-pattern text watermark and keeps the Pattern ID traceable even in the PDF.

## 5. Onboarding Consent UI

Users should explicitly acknowledge the personal-use restriction before generating their first pattern. Do not bury this in a terms scroll.

### 5.1 Where to show it

On Onboarding Slide 3 (the "Real photos at every step" slide), before the "Start creating free →" CTA, add a checkbox:

```
☐  I understand patterns are for personal use only and cannot be sold.
    Full terms →
```

The "Start creating free →" button should be disabled until the checkbox is ticked. Store `termsAcceptedAt` timestamp on the user record in Convex when they tick it.

### 5.2 Convex schema addition

```typescript
// In convex/schema.ts, users table
users: defineTable({
  // ... existing fields
  termsAcceptedAt: v.optional(v.number()),  // Unix timestamp
  termsVersion: v.optional(v.string()),     // e.g. "1.0" — bump when ToS changes
})
```

If `termsVersion` on the user record does not match the current app ToS version, show the consent screen again before the next generation.

## 6. Takedown Process (Operational)

If a pattern generated by the app appears for sale on Etsy or another platform:

1. The Pattern ID in the watermark or PDF footer lets you query your Convex database for the generating account
2. Contact the platform (Etsy, Ravelry, etc.) with:
   - Screenshot of the listing
   - Copy of the pattern showing the embedded Pattern ID and attribution
   - Your Terms of Service showing the personal-use-only restriction
   - Statement that the pattern was generated on your platform and is subject to your ToS
3. Platforms will typically remove the listing within 5-10 business days under their own seller policies, even without a formal copyright claim

You do not need to assert copyright. The ToS violation is sufficient for platform enforcement.
